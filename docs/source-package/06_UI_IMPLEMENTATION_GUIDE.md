# UI Implementation Guide

## 1. 디자인 레퍼런스

`design_reference_매수체크_v3.zip` 안의 HTML 파일을 반드시 확인한다.

주요 레퍼런스:

- `매수리스크체크 v3.dc.html`
- `매수리스크체크.dc.html`
- `부동산비용가이드.dc.html`

최종 방향은 `매수리스크체크 v3.dc.html`을 우선한다.

단, HTML 안의 계산식과 demo data는 프로토타입용이다.  
실제 계산은 규칙 엔진과 DB 기반으로 재구현한다.

## 3. 핵심 화면

### Home

목표: 사용자가 바로 단지를 검색하게 만든다.

구성:

- Header
- Hero copy
- Complex search
- Example search chips
- 신뢰 배지
- 서비스 설명 3단계
- 하단 보조 뉴스레터

### Complex Summary

목표: 단지의 최근 거래 흐름을 사실 기반으로 보여준다.

구성:

- 단지명
- 위치
- 최근 실거래가
- 최근 거래일
- 평단가
- 6개월 변동률
- 거래량
- 전세가율
- Trend chart
- Volume chart
- 외부 매물 보기

금지: 투자 판단, 상승 전망, 추천 표현.

### Condition Form

목표: 사용자의 입력 부담을 줄인다.

원칙:

- 한 번에 하나의 질문
- 선택형 우선
- “잘 모르겠어요” 선택지 제공
- 이전 단계 수정 가능
- 상단 조건 요약

### Report

목표: 1분 안에 핵심 리스크를 이해하게 한다.

구성:

- 총 추가 필요금액
- 매수 조건 요약
- 비용 Breakdown
- Confidence Label
- 적용 가능 규제
- 대출/감면/예외 카드
- 전문가 질문
- 근거 보기
- 외부 매물 링크
- 관심단지 저장
- 공유/PDF 준비 중
- 뉴스레터
- 면책

### Watchlist

목표: 저장한 단지를 다시 확인하게 한다.

구성:

- 저장 단지 카드
- 최근 실거래
- 예상 추가 비용
- 리스크 배지
- 마지막 업데이트일
- 다시 계산
- 비교 추가
- 실제 매물 확인

## 4. 필수 UI State

### Loading

문구가 순차적으로 바뀌어야 한다.

- 최근 실거래가를 확인하는 중입니다
- 규제지역 여부를 확인하는 중입니다
- 예상 세금과 부대비용을 계산하는 중입니다
- 매수 리스크 리포트를 정리하는 중입니다

### Search Empty

- 검색 결과를 찾지 못했어요.
- 단지명이나 동 이름을 조금 다르게 입력해보세요. 현재는 자양동 주요 단지를 먼저 지원하고 있어요.

추천 검색어:

- 자양동
- 래미안
- 더샵
- e편한세상
- 아크로

### Error

- 정보를 불러오지 못했어요.
- 일시적인 연결 문제일 수 있어요. 잠시 후 다시 시도해 주세요.

버튼:

- 다시 시도
- 홈으로 돌아가기

### Watchlist Empty

- 아직 저장한 단지가 없어요.
- 관심 있는 단지를 저장하면 최근 실거래, 예상 비용, 규제 변경을 다시 확인하기 쉬워요.

## 5. Confidence Label UI

라벨:

- 규칙 기반
- 변동 가능
- 별도 확인
- 참고 개념

색상은 muted tone 사용.  
색상만으로 의미를 전달하지 말고 텍스트를 함께 표시한다.

---

## 6. Chart Interaction

Trend chart / Volume chart는 다음을 지원한다.

- hover tooltip
- mobile touch tooltip
- 선택 월 강조
- tooltip에는 월, 평단가, 거래량/거래건수 표시
- 차트 하단 해석 문구 표시

---

## 7. Toast

필수 Toast:

- 관심단지에 저장했어요. 다음에 다시 계산하기 쉽게 보관됩니다.
- 질문을 복사했어요. 중개사나 전문가에게 그대로 물어보세요.
- 구독이 완료됐어요. 중요한 변화가 있으면 쉽게 정리해드릴게요.
- PDF 리포트 기능을 준비 중이에요.

Toast는 2~3초 후 사라지게 한다.

---

## 8. 접근성

- 모바일 터치 영역 44px 이상
- 주요 CTA는 화면당 하나만 가장 강하게
- 텍스트 대비 WCAG AA 수준 목표
- 작은 회색 글씨 남용 금지
- 상태를 색상만으로 전달하지 않기
- focus style 제공
- accordion transition 부드럽게
- 버튼 aria-label 제공
